<objective>
Implement the two independent inspection tools (`get_workload_ir`, `get_accelerator_ir`) with the dual-parameter pattern (experiment_id OR file path) so AI agents can inspect workloads and hardware without running a full solve.

Purpose: These tools let agents explore inputs before committing to an expensive MILP solve. Per D-01, they accept either an experiment_id (to reuse parsed data from a completed job) or a direct file path (for ad-hoc inspection).

Output: Working get_workload_ir and get_accelerator_ir returning Pydantic IR JSON, with structured error handling per D-03.
</objective>
