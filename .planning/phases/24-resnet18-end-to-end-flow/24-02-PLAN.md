**Bug 4 fix (line 224):** Change the strict equality assert to `>=`:

Current:

Replace with:
```python

    f"but node {node.name} needs at least {len(node.tensors)}"

```

This allows the pooling core (which has I1, I2, O = 3 operands) to work with MaxPool (which has 1 input + 1 output = 2 tensors). The existing loop at lines 235-238 already correctly iterates over `range(len(node.inputs))` to match input operands, so only the first N input operands are used. The unused I2 operand is simply not linked.

**Bug 3 fix (get_mapping_attributes, line 243):** The ZigZag `SpatialMappingGeneratorStage` crashes with `ValueError: max() iterable argument is empty` when a core has `served_dimensions: []` on its memory level (`rf_128B` on TPU compute cores) and `SpatialMapping.empty()` is used. The crash happens inside ZigZag's `limit_unrolling_to_mem_capacity()` -> `adjust_unrolling_factors()` when `unrolling_factors` is an empty list.

Add a try/except wrapper around the ZigZag estimation call in the `estimate()` method (line 268) to catch this ZigZag-internal crash and return a fallback cost estimate. This is preferable to modifying ZigZag internals (third-party library).

In the `estimate()` method, replace:
```python
def estimate(self, node: ComputationNode, core: Core) -> CoreCostEntry:
    layer_node = self.get_layer_node(node, core)
    cme = self.run_zigzag(layer_node, core)
    cme = self.increase_cc_per_op(cme, node.type)
    return CoreCostEntry(
        energy_total=getattr(cme, "energy_total", 0),
        latency_total=getattr(cme, "latency_total2", getattr(cme, "ideal_cycle", 0)),
        ideal_cycle=getattr(cme, "ideal_cycle", 0),
        ideal_temporal_cycle=getattr(cme, "ideal_temporal_cycle", 0),
        mem_energy_breakdown=getattr(cme, "mem_energy_breakdown", {}),
        cme=cme,
        mapping=getattr(cme, "mapping", None),
        layer=node,
    )
```

With:
```python
def estimate(self, node: ComputationNode, core: Core) -> CoreCostEntry:
    layer_node = self.get_layer_node(node, core)
    try:
        cme = self.run_zigzag(layer_node, core)
        cme = self.increase_cc_per_op(cme, node.type)
        return CoreCostEntry(
            energy_total=getattr(cme, "energy_total", 0),
            latency_total=getattr(cme, "latency_total2", getattr(cme, "ideal_cycle", 0)),
            ideal_cycle=getattr(cme, "ideal_cycle", 0),
            ideal_temporal_cycle=getattr(cme, "ideal_temporal_cycle", 0),
            mem_energy_breakdown=getattr(cme, "mem_energy_breakdown", {}),
            cme=cme,
            mapping=getattr(cme, "mapping", None),
            layer=node,
        )
    except (ValueError, StopIteration) as e:
        # ZigZag internal crash (e.g., empty unrolling_factors in spatial mapping generation
        # when a core memory level has served_dimensions: [] and the spatial mapping is empty).
        # Fall back to a minimal cost estimate based on ideal computation cycles.
        logger.warning(
            f"ZigZag estimation failed for {node.name} on core {core.id}: {e}. "
            f"Falling back to ideal-cycle estimate."
        )
        # Compute ideal cycles from layer dimensions: product of all dim sizes
        ideal_cycle = 1
        for dim_size in layer_node.layer_dim_sizes.values():
            ideal_cycle *= dim_size
        return CoreCostEntry(
            energy_total=0,
            latency_total=ideal_cycle,
            ideal_cycle=ideal_cycle,
            ideal_temporal_cycle=ideal_cycle,
            mem_energy_breakdown={},
            cme=None,
            mapping=None,
            layer=node,
        )
```

Check that `CoreCostEntry` accepts `cme=None` and `mapping=None`. Read the `CoreCostEntry` dataclass definition in `stream/cost_model/core_cost.py` to verify. If `cme` or `mapping` have non-optional types, change the fallback to pass a compatible default or update the dataclass to accept `None`.
